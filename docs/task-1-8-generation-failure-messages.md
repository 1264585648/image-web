# Task 1.8：生成失败提示优化完成记录

## 本次完成范围

- `/api/generate` 在生成失败时仍会保留 `failed` 任务，前端历史可以继续展示失败原因。
- 后端新增 `_friendly_generation_error()`，把常见底层异常转换成用户可理解的中文提示。
- 统一了部分接口错误文案，包括上传格式、上传图片损坏、原图不存在、模板不存在、任务不存在、下载文件缺失等。
- 生成成功时会清空 `task.error_message`，避免复用旧任务状态时产生误导。

## 已覆盖的错误类型

- 图片无法读取 / 文件损坏。
- 原图文件不存在。
- 服务端写入权限不足。
- 图片过大或内存不足。
- 输出格式不支持。
- 图片尺寸或导出尺寸不符合要求。
- 背景颜色参数不正确。
- 其他低层异常兜底为“换一张更清晰、主体更完整、背景更干净的商品图后重试”。

## 验收建议

1. 上传一张正常 JPG / PNG / WebP，确认仍可生成成功。
2. 删除 `storage/uploads` 中某个源图后请求生成，确认任务进入 `failed`，错误提示为“原图文件不存在，请重新上传商品图后再生成。”
3. 传不存在的 `source_image_id`，确认接口返回 404，提示“原图不存在，请重新上传商品图”。
4. 传错误的 `template_id`，确认接口返回 400，提示“模板不存在，请重新选择模板”。
5. 传错误的输出格式，确认前端或接口不再展示 Python traceback。
