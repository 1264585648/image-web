# 01. 当前能力、主要缺口与目标边界

## 1. 当前已具备的能力

### 1.1 图片处理

当前后端已经具备：

- JPG、PNG、WebP 上传；
- EXIF 方向修正；
- 使用 `rembg` 抠图；
- Alpha 边缘平滑；
- 自动对比度、亮度和锐度优化；
- 商品主体裁剪、缩放和居中；
- 白底、透明底、浅灰底和自定义颜色；
- 轻阴影合成；
- PNG、JPG、WebP 输出；
- 同一任务输出透明图、阴影图和高清图。

### 1.2 基础检测

当前合规模块能够判断：

- 是否检测到商品主体；
- 图片尺寸是否不低于固定阈值；
- 商品是否居中；
- 商品最长边占画布的比例是否在固定范围；
- 背景平均白度是否达到阈值；
- 全图边缘方差是否达到清晰度阈值。

### 1.3 基础产品链路

当前已经有：

- 用户上传；
- 模板选择；
- 后台生成任务；
- 任务进度；
- 多资产结果；
- 历史记录；
- 合规分数和警告信息；
- 结果下载。

## 2. 当前主要问题

## 2.1 检测时机不正确

当前检测发生在白底、居中、缩放之后。由于这些指标由生成逻辑主动控制，检测结果容易出现“系统自己生成、系统自己判高分”的问题。

必须增加独立原图预检，并将原图报告与修复后报告分开保存。

## 2.2 抠图失败会静默降级

当前 `rembg` 异常时会直接保留整张原图继续处理。系统可能把带原始背景的矩形图片放到白色画布中，然后因为外圈画布为白色而误判背景合格。

目标行为：

- 抠图失败必须显式记录；
- 低置信度必须要求人工确认；
- 主体近似整张矩形时不得直接判定白底合格；
- 后续检测必须基于主体 Mask 和置信度。

## 2.3 所有平台共用固定阈值

现有尺寸、占比、居中、背景和清晰度阈值写死在检测函数中。Amazon 主图、Google Merchant 商品图和 Shopify 展示图实际要求不同，不能继续使用一套逻辑。

目标行为：

- 规则由 JSON/YAML 配置加载；
- 支持平台、站点、图片角色和类目过滤；
- 支持规则版本和生效时间；
- 规则新增或调整不要求改动核心检测代码。

## 2.4 背景检测过于粗糙

当前主要使用主体外区域平均亮度，容易漏检：

- 小块灰色或彩色污点；
- 原图矩形边框；
- 背景渐变；
- 抠图残留；
- 白边与彩色光晕；
- 重阴影或倒影。

目标行为：

同时计算纯白像素比例、背景方差、最大异常连通区域、四角一致性、主体周边光晕和边框特征。

## 2.5 主体占比只看最长边

细长商品可能最长边占比很高，但真实主体面积很小。需要同时计算：

- 宽度占比；
- 高度占比；
- 边界框面积占比；
- Mask 实际面积占比；
- 凸包面积占比；
- 四周边距；
- 最窄方向占比。

## 2.6 清晰度检测受背景影响

当前对整张图片做边缘方差，白底面积、商品纹理和噪点都会干扰结果。

目标行为：

- 只对主体 ROI 检测；
- 将模糊、噪点、压缩失真、曝光和过度锐化分开；
- 按检测器给出独立指标，而不是只给“清晰/不清晰”。

## 2.7 缺少高频违规检测

首批缺失能力包括：

- 二维码；
- 条形码；
- 图片边框；
- 商品碰边或裁切；
- 文件扩展名与真实格式不一致；
- 异常旋转；
- 模糊、压缩块、过曝、欠曝；
- 抠图白边、毛边、残留。

第二批缺失能力包括：

- OCR 文字；
- 价格、折扣、Free Shipping 等促销词；
- 水印；
- 后期覆盖 Logo；
- 人物、多目标和多余道具；
- 商品与标题、颜色、数量和套装一致性；
- 内容安全和知识产权风险。

## 3. 产品目标

## 3.1 Compliance MVP

第一阶段必须完成：

1. 原图预检；
2. 抠图状态和 Mask 持久化；
3. 平台规则引擎；
4. 首批 20 条确定性规则；
5. 问题证据和严重程度；
6. 选择性自动修复；
7. 修复后复检；
8. 前后对比；
9. 测试集与指标；
10. 平台实际审核反馈入口。

## 3.2 首发平台

首发只支持：

- `amazon / main / general`；
- `google-merchant / product-image / general`；
- `universal / product-image / general`。

Temu、TikTok Shop、Walmart、eBay 和 Etsy 在首发后按官方资料与真实样本逐步加入。

## 3.3 非目标

以下内容不作为 Compliance MVP 的承诺：

- 保证平台审核通过；
- 覆盖所有类目特殊规则；
- 自动判断所有知识产权问题；
- 自动修改商品颜色、数量、文字和包装；
- 使用单个大模型替代确定性规则；
- 在没有真实审核反馈前声称高准确率。

## 4. 用户体验目标

用户上传图片后，应能看到类似结果：

```text
Amazon 主图预检：未通过

高风险问题
- 背景存在较大非白区域
- 商品右侧触碰画布边缘

质量问题
- 主体区域轻度模糊

可安全修复
- 换纯白背景
- 缩放至推荐占比
- 重新居中
- 导出 2000 × 2000 JPG

需要确认
- 底部透明边缘疑似存在抠图残留
```

修复后：

```text
修复后复检：通过基础规则

已解决 4 项
仍需人工确认 1 项
规则版本：amazon-main-2026.07
```

## 5. 产品文案边界

可以使用：

- 检查常见主图风险；
- 按平台规则生成检测报告；
- 自动修复确定性问题；
- 提高提交前检查效率；
- 帮助降低常见图片拒绝风险。

禁止使用：

- 保证审核通过；
- 100% 合规；
- 官方认证；
- 自动解决全部侵权和平台政策问题；
- 不经确认修改商品真实性信息。
